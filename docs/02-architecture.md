# 2. Многоуровневая архитектура

## Концепция: 4 уровня защиты

Каждый уровень активируется когда предыдущий заблокирован. Клиент автоматически выбирает рабочий.

## Матрица inbound-ов на сервере

| # | Inbound | Port | Transport | SNI/Dest | Назначение |
|---|---------|------|-----------|----------|------------|
| 1 | reality-main | 443 | TCP RAW | www.microsoft.com | Основной |
| 2 | reality-google | 8443 | TCP RAW | dl.google.com | Резервный SNI |
| 3 | reality-apple | 2053 | TCP RAW | www.apple.com | Ещё один SNI |
| 4 | ws-cloudflare | 2082 | WebSocket | (через CF домен) | CDN-фронтинг |
| 5 | relay-xhttp | 10443 | xHTTP | www.microsoft.com | Relay Layer 2 |
| 6 | grpc-transport | 10444 | gRPC | updates.microsoft.com | Ещё один транспорт |

## Компоненты

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| Сервер | Xray-core v26.3.27 | Прокси-сервер с VLESS Reality |
| Панель управления | 3X-UI v2.8.11 | Веб-интерфейс для управления |
| CDN | Cloudflare (бесплатно) | Фронтинг для обхода белых списков |
| SSL | Let's Encrypt (IP) | Шифрование панели управления |
| Мониторинг | monitor-xray.sh + cron | Автоперезапуск при крашах |
| Клиент Windows | Hiddify 4.1.1 | VPN-клиент с GUI + split routing |
| Клиент Android | v2rayNG | Мобильный VPN-клиент + split routing |
| Клиент iOS | Shadowrocket ($2.99) | Лучший iOS-клиент + split routing |
| Управление | ssh_exec.py | SSH-утилита для управления сервером |

## Приоритет при подключении (auto-select)

1. **reality-main:443** — если работает, используем (минимальная латентность)
2. **reality-google:8443** — если 443 заблокирован оператором
3. **reality-apple:2053** — ещё один резервный SNI
4. **ws-cloudflare:2082** — если все прямые подключения к VPS заблокированы
5. **Relay через РФ** — если IP VPS в чёрном списке целиком
6. **WebRTC** — аварийный вариант при полном white-list
