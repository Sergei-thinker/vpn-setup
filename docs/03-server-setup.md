# 3. Настройка сервера

## Шаг 1: Аренда VPS

**Рекомендуемые провайдеры:** AlphaVPS, RackNerd, Hetzner, Netcup — любой EU KVM VPS с Debian 12.

> **⛔ Не используйте российские хостинги** (Aeza, VDSina, REG.RU) — подчиняются Роскомнадзору.

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| CPU | 1 core | 1-2 core |
| RAM | 512 MB | 1-2 GB |
| Диск | 5 GB | 10+ GB |
| Канал | 100 Mbit/s | 1 Gbit/s |
| Трафик | 1 TB | Безлимитный |
| Цена | ~$1-2/мес | ~$3-5/мес |
| Локация | Швеция (Стокгольм) |
| ОС | Debian 12 |

**Почему этот тариф:**
- Самый дешёвый на рынке (1.99 EUR vs 5-10 EUR у конкурентов)
- 4 GB RAM — с запасом для Xray (нужно ~256 МБ)
- Безлимитный трафик — не нужно следить за объёмом
- Швеция — приемлемый пинг из РФ (~50-70 мс)
- Почасовая оплата — можно пересоздать сервер если IP заблокирован

**Ограничения промо-тарифа (не критичны):**
- Нет техподдержки
- Канал до 300 Мбит/с (достаточно для любых задач)
- Нет дополнительных IPv4
- IOPS до 1000 (VPN почти не нагружает диск)

**Альтернативные провайдеры:**

| Провайдер | Цена | Плюсы | Минусы |
|-----------|------|-------|--------|
| 4VPS (4vps.su) | от 80 RUB/мес | Бюджетный, много локаций | — |
| is*hosting | от 300 RUB/мес | Крипто-оплата, 40+ ДЦ | Дороже |
| Fornex | от 300 RUB/мес | Стабильные "чистые" IP | Дороже |
| UFO.Hosting | от 200 RUB/мес | 45+ стран, безлимит | — |

**Не рекомендуются:** Hetzner, OVH, DigitalOcean, AWS — подсети массово блокируются ТСПУ.

## Шаг 2: Базовая настройка сервера

### Подключение по SSH
```bash
ssh root@<VPS_IP>
```

### Обновление системы
```bash
apt update && apt upgrade -y
```

### Установка 3X-UI панели
```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

При установке скрипт спрашивает:
1. **Panel Port** — задать вручную (введи `y`, затем порт, например `2053`). Если пропустить — сгенерирует случайный
2. **Username / Password** — логин и пароль для веб-панели
3. **SSL Certificate** — выбрать опцию `2` (Let's Encrypt for IP Address)
4. **IPv6** — пропустить (Enter)
5. **ACME port** — оставить 80 (Enter)

### Управление панелью
```bash
x-ui start          # Запуск
x-ui stop           # Остановка
x-ui restart        # Перезапуск
x-ui status         # Статус
x-ui settings       # Текущие настройки (порт, URL)
x-ui setting -username USER -password PASS   # Сменить логин/пароль
```

## Шаг 3: Настройка VLESS Reality

### Вход в панель
Открыть в браузере URL из `x-ui settings`, например:
```
https://<VPS_IP>:<PANEL_PORT>/<PANEL_PATH>/
```

### Создание Inbound
1. Перейти в **Inbounds** -> **+ Add Inbound**
2. Заполнить форму:

| Поле | Значение | Пояснение |
|------|----------|-----------|
| Enabled | Включено | — |
| Remark | `reality` | Любое имя |
| Protocol | `vless` | — |
| Port | `443` | Стандартный HTTPS-порт |
| Transmission | `TCP (RAW)` | — |
| Security | `Reality` | Включает маскировку |
| uTLS | `chrome` | Имитация браузера Chrome |
| Target | `www.microsoft.com:443` | Сайт для маскировки |
| SNI | `www.microsoft.com` | Домен в Server Name Indication |

3. Раскрыть секцию **Client**:
   - **Flow:** `xtls-rprx-vision`

4. Нажать **"Get New Cert"** — сгенерирует Public Key и Private Key

5. Нажать **Create**

### Получение ссылки/QR
- В списке Inbounds нажать `...` -> QR Code / Export Link
- **Правый QR** (reality-xxx) — конфиг для одного сервера
- **Левый QR** (Subscription) — подписка с автообновлением
- Ссылка формата: `vless://UUID@IP:443?...`

### Критически важные замечания
- **Xray-core >= 25.12.8** — старые версии детектируются через TLS 1.3 NewSessionTicket
- **SNI** — выбрать сайт с TLS 1.3 и H2, IP которого близок к IP вашего VPS
- **Время на устройстве** — должно быть автоматическим, расхождение >30 сек = handshake не пройдёт
